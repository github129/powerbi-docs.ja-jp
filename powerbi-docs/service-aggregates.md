# <a name="work-with-aggregates-sum-average-and-so-on-in-the-power-bi-service"></a>Power BI サービスでの集計 (sum、average、) を使用します。

## <a name="what-is-an-aggregate"></a>集計とは

データの値を数学的に結合したい場合があります。 数学的演算には、合計、平均値、最大、カウント、およびなどができます。 データ値を結合するときに呼び出された*集計*します。 その数学的演算の結果は "*集計値*" です。

Power BI サービスと Power BI Desktop は、視覚エフェクトを作成する際にデータを集計する場合があります。 多くの場合、必要なのは集計のみですが、別の方法で値を集計する場合もあります。  たとえば、合計と平均について考えてみます。 管理して、視覚エフェクトで Power BI を使用して集計を変更するいくつかのさまざまな方法はあります。

最初に、データを見ていきましょう*型*のため、Power BI と集計かどうか、および方法については、データの種類を決定します。

## <a name="types-of-data"></a>データの型

ほとんどのデータセットには複数のデータ型があります。 最も基本的なレベルでデータが数値またはありません。 Power BI では、数値を合計、平均、カウント、最小、差異、および多くを使用してデータを集計できます。 サービスは、多くの場合と呼ばれる、テキスト形式のデータを集計できますも*カテゴリ*データ。 ように数値のみのバケットに配置することによって、カテゴリ別フィールドを集計しようとする場合**値**または**ツールヒント**、Power BI は各カテゴリの出現回数をカウントまたはそれぞれの別個の発生をカウントカテゴリ。 特殊な種類の日付のように、データがある独自の集計オプションのいくつか: 最も早い、最終、最初と最後です。

## <a name="why-dont-aggregates-work-the-way-i-want-them-to"></a>集計が思い通りに動作しないのはなぜですか?

Power BI での集計を使用するサービスは混乱を招くできます。 数値フィールドがあるかもしれませんし、Power BI には、集計を変更することはできません。 それとも、年度のようなフィールドがあるとき、集計せず、発生数だけを数えたいのでしょうか。

通常、根本的な問題は、データセットのフィールドの定義です。 データセットの所有者がテキストとして、フィールドを定義する可能性がありますし、Power BI の合計または平均できない理由を説明します。 残念ながら、[フィールドの分類方法を変更できるのはデータセットのオーナーだけです](desktop-measures.md)。 したがって Desktop またはデータセット (Excel など) を作成するために使用するプログラムは、データセットに対する所有者アクセス許可がある場合は、この問題を解決できます。 それ以外の場合は、支援を得るためにデータセットの所有者に連絡する必要があります。  

この記事の最後に特別なセクションがある[**考慮事項とトラブルシューティング**](#considerations-and-troubleshooting)します。 ヒントとガイダンスを提供します。 そこで回答が見つからない場合に、質問を投稿、 [Power BI コミュニティ フォーラム](http://community.powerbi.com)します。 Power BI チームから直接、迅速な応答を取得します。

## <a name="considerations-and-troubleshooting"></a>考慮事項とトラブルシューティング

質問: **[集計しない]** オプションが表示されないのはなぜですか。

回答:選択しているフィールドが、Excel または [Power BI Desktop](desktop-measures.md) で作成された計算メジャーまたは詳細メジャーである可能性があります。 それぞれの計算メジャーには、独自のハードコーディングされた式があります。 Power BI を使用して集計を変更することはできません。 たとえば、それが合計である場合、使用できるのは合計のみとなります。 **フィールド**一覧が表示されます*計算メジャー*計算機の記号。

Q:フィールドは**数値**ですが、選択肢が **[カウント]** と **[個別のカウント]** だけなのはなぜですか?

回答 1:考えられる説明は、データセットの所有者は*いない*フィールドを数値として分類されます。 たとえば、データセットがある場合、**年**フィールドでは、データセットの所有者がテキストとして値を分類する可能性があります。 Power BI がカウントされる可能性が高く、**年**フィールド (たとえば、1974 年に生まれた人の数)。 Power BI が合計または平均には小さい可能性があります。 所有者である場合は、Power BI Desktop でデータセットを開き、使用して、**モデリング**タブ データ型を変更します。

回答 2:フィールドに計算機アイコンがある場合はつまり、*計算されるメジャー*します。 それぞれの計算メジャーのデータセットの所有者のみが変更できる、独自のハードコーディングされた式があります。 Power BI を使用して、計算は平均や合計のような単純な集計にあります。 「合計開始から年の」、"percent 親カテゴリに対する貢献の"のような複雑なもの、または場合もあります。 Power BI は、合計や平均結果をわけです。 代わりに、それだけ再計算されます (ハードコーディングされた式を使用して) 各データ ポイント。

回答 3:別の可能性としては、フィールドが "*バケット*" に配置されています。その場合、カテゴリ値のみが許可されます。  選択肢は、カウントと個別のカウントだけになります。

回答 4:4 番目の方法は、軸に対してフィールドを使用していること。 たとえば、Power BI では、棒グラフの軸で、個別の値ごとに棒が 1 本表示されます。フィールド値はまったく集計されません。

>[!NOTE]
>このルールの例外は散布図です。X 軸と Y 軸に集計値が*要求されます*。

A:SSAS 多次元モデルにライブ接続すると、クライアント側で集計できません (first、last、avg、min、max、sum を含む)。
